### 🎯 可复用经验总结

#### 1. 技术方案复用
##### ✅ Python+JavaScript模板转义最佳实践
```javascript
// 问题：Python f-string会解析${}和反斜杠，导致JavaScript代码语法错误
// 解决方案：双重转义规则
text = text.replace(/\\*\\*(.*?)\\*\\*/g, '<strong>$1</strong>');  // 正则反斜杠双写
const apiKey = `${{ QWEN_API_KEY }}`;  // JS变量用${{}}包裹避免被Python解析
```
**适用场景：** 所有在Python后端拼接前端HTML/JS代码的场景，可避免90%的语法解析错误。

##### ✅ 视频截图功能实现方案
```javascript
function takeScreenshot() {
    const video = document.querySelector('video');
    const canvas = document.createElement('canvas');
    canvas.width = video.videoWidth;
    canvas.height = video.videoHeight;
    canvas.getContext('2d').drawImage(video, 0, 0, canvas.width, canvas.height);
    return canvas.toDataURL('image/png');
}
```
**适用场景：** 任意HTML5 Video元素的截图功能，无需第三方依赖，纯前端实现。

##### ✅ localStorage持久化通用模板
```javascript
// 初始化读取
document.addEventListener('DOMContentLoaded', () => {
    const saved = localStorage.getItem('storageKey');
    if (saved) data = JSON.parse(saved);
});

// 保存函数
function saveToLocalStorage() {
    localStorage.setItem('storageKey', JSON.stringify(data));
}
```
**适用场景：** 所有需要本地持久化的前端数据（笔记、用户配置、历史记录等），无需后端存储。

#### 3. 调试经验复用
##### ✅ 多层转义错误定位方法
当遇到浏览器JS语法错误但代码看起来正确时：
1. 先查看页面源代码，确认Python输出的最终代码是什么
2. 检查是否有反斜杠、${}等被Python解析掉的情况
3. 逐行对比源代码和输出代码，定位转义缺失位置

##### ✅ 依赖冲突快速排查
端口占用：`netstat -ano | findstr :9000` → 找到PID → 杀死进程
依赖缺失：`pip install -r requirements.txt` 一次性安装所有依赖
